API文档概览

本文档提供了树状网络文件系统（Dendrachy Network File-System, dnfs）的所有API端点说明。 这些API允许您与系统进行交互，包括用户认证、笔记管理、AI服务和传感器数据操作。

认证API

用户注册、登录和令牌管理

笔记API

笔记的创建、查询、更新和删除

AI服务API

DeepSeek大模型调用和文档生成

传感器API

传感器数据的接收和查询

API使用说明

大部分API端点需要认证，需要在请求头中包含有效的JWT访问令牌：Authorization: Bearer {access_token}

访问令牌有效期为1小时，过期后需要使用刷新令牌获取新的访问令牌。

所有API请求和响应使用JSON格式，除非特别说明。

认证API

POST /register 用户注册

描述

创建新用户账户

请求参数

参数名 类型 是否必需 描述
username string 必需 用户名
password string 必需 密码

请求示例

{"username": "admin", "password": "your_password"}

响应示例

成功 (201 Created)

{"message": "注册成功"}

失败 - 参数错误 (400 Bad Request)

{"error": "用户名和密码不能为空"}

失败 - 用户已存在 (400 Bad Request)

{"error": "用户名已存在"}
POST /login 用户登录

描述

用户登录并获取访问令牌和刷新令牌

请求参数

参数名 类型 是否必需 描述
username string 必需 用户名
password string 必需 密码

请求示例

{"username": "admin", "password": "your_password"}

响应示例

成功 (200 OK)

{"access_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...", "refresh_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9..."}

失败 - 参数错误 (400 Bad Request)

{"error": "用户名和密码不能为空"}

失败 - 认证失败 (401 Unauthorized)

{"error": "用户名或密码错误"}
GET /profile 获取当前用户信息

描述

获取当前登录用户的信息

请求头

Authorization: Bearer {access_token}

响应示例

成功 (200 OK)

{"username": "admin"}
POST /refresh 刷新访问令牌

描述

使用刷新令牌获取新的访问令牌

请求参数

参数名 类型 是否必需 描述
refresh_token string 必需 刷新令牌

请求示例

{"refresh_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9..."}

响应示例

成功 (200 OK)

{"access_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9..."}

失败 - 参数错误 (400 Bad Request)

{"error": "刷新令牌不能为空"}

失败 - 令牌无效 (401 Unauthorized)

{"error": "无效的刷新令牌"}

笔记API

POST /add_note 添加笔记

描述

添加一条新的笔记

请求头

Authorization: Bearer {access_token}

请求参数

参数名 类型 是否必需 描述
pid integer 必需 父笔记ID，根笔记为0
title string 必需 笔记标题
content string 必需 笔记内容

请求示例

{"pid": 0, "title": "测试笔记", "content": "这是一条测试笔记的内容"}

响应示例

成功 (200 OK)

{"message": "Note added successfully"}

失败 - 参数错误 (400 Bad Request)

{"error": "Pid, title and content are required"}
POST /get_notes 获取笔记

描述

获取指定父笔记ID的所有子笔记

请求头

Authorization: Bearer {access_token}

请求参数

参数名 类型 是否必需 描述
pid integer 可选 父笔记ID，默认值为0

请求示例

{"pid": 0}

响应示例

成功 (200 OK)

[{"id": 1, "pid": 0, "title": "测试笔记", "content": "这是一条测试笔记的内容"}, {"id": 2, "pid": 0, "title": "另一条笔记", "content": "内容"}]
POST /delete_note 删除笔记

描述

删除指定ID的笔记

请求头

Authorization: Bearer {access_token}

请求参数

参数名 类型 是否必需 描述
id integer 必需 笔记ID

请求示例

{"id": 1}

响应示例

成功 (200 OK)

{"message": "Note deleted successfully"}

失败 - 参数错误 (400 Bad Request)

{"error": "Note ID is required"}

失败 - 笔记不存在 (404 Not Found)

{"error": "Note not found"}
POST /update_note 更新笔记

描述

更新指定ID笔记的内容

请求头

Authorization: Bearer {access_token}

请求参数

参数名 类型 是否必需 描述
id integer 必需 笔记ID
pid integer 必需 父笔记ID
title string 必需 笔记标题
content string 必需 笔记内容

请求示例

{"id": 1, "pid": 0, "title": "更新后的笔记标题", "content": "更新后的笔记内容"}

响应示例

成功 (200 OK)

{"message": "Your change is up-to-date."}

失败 - 参数错误 (400 Bad Request)

{"error": "Invalid param"}

失败 - 笔记不存在 (404 Not Found)

{"error": "Note not found"}

AI服务API

POST /api/deepseek 调用DeepSeek大模型

描述

调用DeepSeek大模型并返回流式响应，支持文档生成工具调用

请求头

Authorization: Bearer {access_token}

请求参数

参数名 类型 是否必需 描述
prompt string 必需 对话提示词

请求示例

{"prompt": "请帮我写一份项目计划书"}

响应格式

流式响应 (text/event-stream)，每行一个JSON对象：

文本响应

{"content": "生成的文本内容..."}

文档生成通知

{"action_required": "download_document", "document_id": "abc123", "message": "文档已生成，请点击下载按钮获取"}

错误响应

{"error": "错误信息"}
GET /api/download-document/<doc_id> 下载生成的文档

描述

下载由DeepSeek生成的文档

路径参数

参数名 类型 是否必需 描述
doc_id string 必需 文档ID（从/api/deepseek响应中获取）

响应

返回文本文件 (text/plain)，文件名中包含时间戳

POST /api/generate-document 生成文档

描述

调用DeepSeek大模型生成结构化JSON文档并提供下载

请求头

Authorization: Bearer {access_token}

请求参数

参数名 类型 是否必需 描述
prompt string 必需 文档生成提示词

请求示例

{"prompt": "请生成一份关于物联网安全的详细报告"}

响应

成功时返回文本文件 (text/plain)，包含JSON格式的文档内容

传感器API

POST /api/sensor-data 接收传感器数据

描述

接收来自微控制器的传感器数据并保存到数据库

请求参数 - 单条数据格式

参数名 类型 是否必需 描述
device_id string 必需 设备标识符
sensor_type string 必需 传感器类型
value number 必需 传感器值
timestamp string 可选 时间戳 (YYYY-MM-DD HH:MM:SS)

请求参数 - 批量数据格式

参数名 类型 是否必需 描述
device_id string 必需 设备标识符
data array 必需 传感器数据数组，每个元素包含sensor_type、value和可选的timestamp

请求示例 - 单条数据

{"device_id": "esp32_1", "sensor_type": "temperature", "value": 25.5, "timestamp": "2023-05-15 14:30:00"}

请求示例 - 批量数据

{"device_id": "esp32_1", "data": [{"sensor_type": "temperature", "value": 25.5}, {"sensor_type": "humidity", "value": 60.2}]}

响应示例

成功 - 单条数据 (200 OK)

{"message": "传感器数据保存成功"}

成功 - 批量数据 (200 OK)

{"message": "成功保存 2 条传感器数据", "saved": 2}

失败 - 参数错误 (400 Bad Request)

{"error": "缺少device_id字段"}
GET /api/sensor-data 获取传感器数据

描述

获取传感器数据，支持按设备、传感器类型、时间范围过滤

请求头

Authorization: Bearer {access_token}

查询参数

参数名 类型 是否必需 描述
device_id string 可选 设备标识符
sensor_type string 可选 传感器类型
start_time string 可选 开始时间 (ISO格式)
end_time string 可选 结束时间 (ISO格式)
limit integer 可选 返回数据条数限制，默认100

请求示例

GET /api/sensor-data?device_id=esp32_1&sensor_type=temperature&start_time=2023-05-15T00:00:00Z&limit=50

响应示例

[{"id": 1, "device_id": "esp32_1", "sensor_type": "temperature", "value": 25.5, "timestamp": "2023-05-15T14:30:00+08:00"}, {"id": 2, "device_id": "esp32_1", "sensor_type": "temperature", "value": 26.0, "timestamp": "2023-05-15T14:40:00+08:00"}]
