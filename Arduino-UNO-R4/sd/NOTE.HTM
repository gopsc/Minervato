Arduino Mozart API 文档

文件操作 API

POST/api/create

创建空白文件或覆盖现有文件为空白
{"filename": "[filename]"}
filename: 要创建的文件名（必填）

POST/api/append

在文件末尾追加内容
{"filename": "[filename]", "content": "[content]"}
filename: 目标文件名（必填）
content: 要追加的内容（必填）

POST/api/upload

保存或覆盖整个文件内容（适用于较小文件）
{"filename": "[filename]", "content": "[content]"}
filename: 目标文件名（必填）
content: 文件内容（必填）

DELETE/api/remove

删除指定文件
{"filename": "[filename]"}
filename: 要删除的文件名（必填）

传感器与命令 API

POST/api/show

展示所有可用的命令功能
无需参数

POST/api/sensor

执行传感器命令
{"cmd": "[command]"}
cmd: 要执行的命令，如 MPU6050、BME280 等（必填）

POST/api/sensors

执行存储在SD卡上的脚本
{"path": "[script_path]"}
path: SD卡上的脚本文件路径（必填）

GET 请求

直接访问文件或目录：
GET /[filename] 或 GET /[directory]
- 文件会直接返回其内容
- 目录会返回其内容列表

使用示例

创建文件并写入内容（分段）

// 步骤1：创建空白文件
POST /api/create
{"filename": "example.txt"}

// 步骤2：追加内容
POST /api/append
{"filename": "example.txt", "content": "第一段内容\n"}

// 步骤3：继续追加内容
POST /api/append
{"filename": "example.txt", "content": "第二段内容"}

获取状态信息

POST /api/show
{}

说明：获取系统状态信息。不需要任何参数，直接返回状态消息。

创建空白文件

POST /api/create
{"filename": "newfile.txt"}

说明：创建或覆盖为空白文件。如果文件已存在，会先删除再创建。

追加文件内容

POST /api/append
{"filename": "file.txt", "content": "要追加的内容"}

说明：在现有文件末尾追加内容。如果文件不存在，会自动创建。

读取传感器数据

POST /api/sensor
{"cmd": "MPU6050"}

说明：发送传感器命令并获取数据。cmd参数指定要执行的传感器命令。

列出文件夹内容

POST /api/list
{"path": ""}

说明：列出指定路径下的所有文件和目录。返回JSON数组，每个元素包含name(文件名)、isDirectory(是否为目录)和可选的size(文件大小)字段。

上传/保存文件

POST /api/upload
{"filename": "test.txt", "content": "文件内容"}

说明：保存或覆盖文件。如果文件已存在，会先删除再创建。返回保存状态和写入字节数。

通过脚本获取传感器数据

POST /api/sensors
{"path": "script.js"}

说明：执行指定路径的JavaScript脚本获取传感器数据。需要SD卡和JS解释器支持。
